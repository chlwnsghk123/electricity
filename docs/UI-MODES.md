# UI · 화면 · 모드 (UI-MODES)

## 1. 화면(View) 5종

| view | 섹션 ID | 용도 |
|---|---|---|
| `home` | `#home` | 회차 선택. 우측 진행률 (`64/100` 등) |
| `modes` | `#modes-view` | 학습 / 랜덤 학습 / 모의고사 카드 3개 |
| `list` | `#list-view` | 학습 모드 전용 — 과목 탭 + 검색 + 필터 칩 + 카드 리스트 |
| `detail` | `#detail-view` | 문제 상세 (선택지, 정답 공개, 태그, 메모, AI) |
| `result` | `#result-view` | 모의고사 채점 결과 |

전환은 `show(view)` 한 곳에서. 진입할 때마다 해당 `render*()` 호출.

## 2. 모드(Mode) 3종

### 2.1 study (학습 모드)
- list → detail → list
- 답은 `tempAnswer`에 메모리 보관, 다른 문제로 이동/뒤로 가면 소멸
- 답 선택 즉시 정·오 색 표시 + 자동 태그 (green/red)
- prev/next 는 현재 필터(`state.filteredNos`) 기준

### 2.2 random (랜덤 학습)
- 시작 시 과목 선택 시트 → 첫 랜덤 문제 → 바로 detail
- "다음 →" 버튼이 "다음 랜덤 →"으로. prev 비활성.
- 같은 과목 내 미출제 풀에서 랜덤 1개. 다 보면 풀 리셋 + 토스트 알림.
- 답 처리/자동 태그는 study와 동일.

### 2.3 exam (모의고사)
- 진입 시 정식(100문제·150분) / 하프(50문제·75분) 선택 시트
- 각 과목에서 셔플 후 상위 N문제 픽 (정식 N=20, 하프 N=10) → `state.examSet`
- 리스트 화면 건너뛰고 바로 첫 문제 detail
- 헤더 우측에 ⏱ 타이머 표시 (10분 이하 시 경고색)
- 정·오 색 미표시. 답은 `examAnswers`에 보관.
- 마지막 문제 next 버튼이 "시험 제출하기"로 변환
- 제출 시 `grade()` → result + green/red 일괄 자동 태그
- 시간 만료 시 자동 제출 (`tickTimer`)
- "뒤로" 누르면 시험 중단 확인 모달

## 3. 상세 화면 구성

```
[← 뒤로]  문제 N  [⏱ 타이머]  [● 태그트리거]  [☆ 북마크]
─────────────────────────────────────────────────────
[과목] No. N  [진행도(모의/랜덤만)]
문제 본문 (KaTeX 렌더)
[본문 이미지 — <no>.png]
[image_note 캡션]
─ 선택지 ────────────────────────────
① 텍스트                    [<no>-1.png]
② ...                       [<no>-2.png]
③ ...                       [<no>-3.png]
④ ...                       [<no>-4.png]
─ 정답 공개 (학습/랜덤 + 답 선택 후) ─
정답: ② · 내 답: ③ (오답)
─ AI ────────────────────────────────
[ AI에게 질문 → ]
─ 메모 ──────────────────────────────
내 메모  3개          [＋]
(꾹 누르거나 우클릭으로 메모 삭제)
- AI 답변 저장 ...
- 직접 작성 ...
─────────────────────────────────────
[← 이전]                [다음 → / 시험 제출하기]
```

## 4. 태그 (6색 + 의미)

| 색 | 의미 | 자동? |
|---|---|---|
| gray   | 미완료 / 지우기 | 기본값 (실제 저장은 안 됨) |
| green  | 맞춤            | 학습/랜덤 정답 시 / 모의고사 제출 시 |
| red    | 틀림            | 학습/랜덤 오답 시 / 모의고사 제출 시 |
| orange | 복습 필요       | 수동 |
| yellow | 개념 복습 필요  | 수동 |
| blue   | 중요            | 수동 |

UX
- 상세 **메타줄**(과목 배지 · No.) 오른쪽에 인라인 태그 칩 → 클릭 시 6색 팝오버
- 태그가 지정되면 칩 자체가 해당 색 톤 + 의미 문구로 강조 (예: 🔴 "틀림")
- 같은 색 재선택 또는 gray 선택 → 해제
- 학습 모드 리스트의 필터 칩으로 색별 필터 가능 (gray 제외)
- 학습 리스트 카드 우측에도 태그 의미 텍스트 라벨이 직접 노출 (이전의 📷/∑ 아이콘은 제거)
- "북마크" 필터도 별개로 존재

## 5. 메모

- 상세 하단 섹션. AI 답변에 "📝 메모에 저장" 버튼이 있어 한 번 클릭으로 적재
- 헤더 `＋` 버튼 → 인라인 textarea (Ctrl+Enter 저장)
- 삭제: 각 메모 아이템을 **꾹 누르기(600ms)** 또는 **우클릭** → 확인 → 삭제
- 누르는 동안 붉은 하이라이트 피드백

## 6. 바텀시트 4종

| ID | 용도 |
|---|---|
| `#ai-sheet` | AI Q&A. 문제 전문(+본문 이미지) 컨텍스트 + 펼치기/접기 + 추천 질문 + 메시지 + 입력. 추천 질문 클릭 시 즉시 전송. 닫아도 대화는 문제별로 localStorage에 보존되어 다시 열면 복구. 답변 도착 시 답변 시작점으로 스크롤. |
| `#settings-sheet` | 야간 모드 토글 + 진행도 초기화 (확인 모달 동반) |
| `#choice-sheet` | 모의고사 유형 / 랜덤 학습 과목 선택 등 동적 항목 |
| `#tag-popover` | 6색 드롭다운 (시트가 아닌 floating panel). 상세 메타줄의 인라인 태그 칩을 기준으로 위치 |

## 8. 반응형 / 색상

- 전역 토큰은 `styles.css` `:root`에 정의 (Toss 라이트 테마)
- 6색 태그: `--tag-{gray|green|red|orange|yellow|blue}`
- 모바일 우선 (`max-width: 640px` 중앙 정렬, 100dvh)
- 좁은 화면(`@media (max-width: 380px)`)에서 폰트 다운, 태그 피커 1열로
